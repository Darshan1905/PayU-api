Architecture

Middleware runs on connect.pavokart.com (AWS Lightsail). PayU Merchant Key + Salt live on pavokart.com WordPress plugin. Middleware calls WordPress REST, which calls PayU production API.

Client → GET  https://connect.pavokart.com/api/payu/products
       → WooCommerce catalog (simple + variable products)

Client → POST https://connect.pavokart.com/api/payu/create-order (lineItems + customer)
       → https://pavokart.com/wp-json/payu/v1/create-order
       → WooCommerce order created (products + qty, no shipping)
       → https://api.payu.in/v2/payments
       → checkoutUrl returned — redirect customer to PayU hosted page
       → On payment success: WooCommerce order marked paid, stock reduced

Amount-only (legacy): POST /api/payu/initiate — no WooCommerce order

Authentication

Demo API Key (after php artisan db:seed): payu-pavokart-demo-key

HeaderValue
Content-Typeapplication/json
Acceptapplication/json
X-API-Keypayu-pavokart-demo-key (or your client key)

Alternatives: Authorization: Bearer <key> or api_key in JSON body.

IP whitelist: Only IPs listed in API_ALLOWED_IPS (Laravel .env on Lightsail) can call /api/payu/*. Add each client server's public IP. If you get respCode: 4030, your IP is not whitelisted.

POSTCreate Order + Payment Link (recommended)

POST https://connect.pavokart.com/api/payu/create-order

Creates a real WooCommerce order on pavokart.com, then returns PayU checkoutUrl. All line items are validated first — if any product is invalid or out of stock, the entire cart fails (no partial orders). Amount is calculated server-side from product prices. No shipping line is added.

FieldRequiredDescription
lineItemsYesArray of { productId, quantity, variationId? }
lineItems[].productIdYesWooCommerce product ID
lineItems[].quantityYesMin 1
lineItems[].variationIdConditionalRequired for variable products
customerNofirstName, lastName, email, phone, address1, city, state, zipCode, country
callbackUrlConditionalRequired unless set on client API key record

PayU Webhook (middleware)

POST https://connect.pavokart.com/webhook/payu

Use this URL as callbackUrl when initiating payments via the middleware. PayU / WordPress relays payment result here. No X-API-Key required on this route.

In proxy mode, PayU may also callback WordPress first at:

  • https://pavokart.com/wp-json/payu/v1/callback/success
  • https://pavokart.com/wp-json/payu/v1/callback/failure

WordPress Plugin Setup (pavokart.com)

  1. Install payu-payment-gateway plugin on pavokart.com
  2. WooCommerce → PayU Hosted → enter Merchant Key + Salt
  3. Set Middleware proxy secret (same as PAYU_PROXY_SECRET in Laravel .env)
  4. Enable PayU Hosted under WooCommerce → Payments
WP REST endpointPurpose
https://pavokart.com/wp-json/payu/v1/initiate-paymentMiddleware proxy initiate
https://pavokart.com/wp-json/payu/v1/statusMiddleware proxy status
https://pavokart.com/wp-json/payu/v1/callback/successPayU success callback (WooCommerce)
https://pavokart.com/wp-json/payu/v1/callback/failurePayU failure callback (WooCommerce)

Server Configuration (Lightsail)

Example .env values on connect.pavokart.com:

APP_URL=https://connect.pavokart.com

API_IP_WHITELIST_ENABLED=true
API_ALLOWED_IPS=127.0.0.1,::1,<client-server-ip>

PAYU_PROXY_URL=https://pavokart.com
PAYU_PROXY_SECRET=<same-as-wp-plugin-middleware-secret>
PAYU_CALLBACK_URL=https://connect.pavokart.com/webhook/payu

After changing .env, run: php artisan config:clear
