PayU Hosted Checkout Middleware API

PayU v2 Hosted Checkout (non-seamless) with WordPress proxy on pavokart.com

Version1.1.0
Local Base URLhttp://127.0.0.1:8000
Auth HeaderX-API-Key
WordPress Proxyhttps://pavokart.com

Architecture

Middleware runs on your server (local / AWS Lightsail). PayU credentials live on pavokart.com WordPress plugin. Middleware calls WordPress REST, which calls PayU production API.

Client → POST /api/payu/initiate (X-API-Key)
       → https://pavokart.com/wp-json/payu/v1/initiate-payment
       → https://api.payu.in/v2/payments
       → checkoutUrl returned — redirect customer to PayU hosted page

Authentication

Demo API Key (after php artisan db:seed): payu-pavokart-demo-key

HeaderValue
Content-Typeapplication/json
X-API-Keypayu-pavokart-demo-key

Alternatives: Authorization: Bearer <key> or api_key in JSON body.

IP whitelist: Set API_ALLOWED_IPS in Laravel .env. For local Postman use 127.0.0.1,::1. On AWS Lightsail add your client server public IP.

PayU Webhook (middleware)

POST /webhook/payu

Register this public URL in PayU callback settings when using direct mode. In proxy mode, PayU callbacks hit WordPress first; middleware receives relay when configured.

No X-API-Key on this route.

WordPress Plugin Setup (pavokart.com)

  1. Install payu-payment-gateway plugin
  2. WooCommerce → PayU Hosted → enter Merchant Key + Salt
  3. Set Middleware proxy secret (same as PAYU_PROXY_SECRET in Laravel)
  4. Enable PayU Hosted under WooCommerce → Payments
