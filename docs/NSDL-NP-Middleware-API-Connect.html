NSDL NP Middleware API
Production reference — connect.striveedgepaints.com

1) Initiate payment

Purpose: Start a collect / obtain payment links.

Method / URL

POST https://connect.striveedgepaints.com/api/nsdl-np/initiate

Headers

HeaderValue
Content-Typeapplication/json
Acceptapplication/json
X-API-Keynsdl-np-demo-local-key (or your client key)

Body fields (JSON)

FieldRequiredNotes
requestAmountYesNumber ≥ 0.01
collect_refNoMax 30 chars; omit to let upstream generate
user_refNoMax 50 chars
display_nameNoMax 200 chars
transaction_noteNoMax 500 chars
remarksNoMax 500; used if transaction_note empty

Example body

{
  "collect_ref": "TESTREF001",
  "requestAmount": "10.50",
  "user_ref": "custuser01",
  "display_name": "Test Customer",
  "transaction_note": "Order note",
}

Success response (HTTP 200)

{
  "status": true,
  "respCode": 2000,
  "respMessage": "Payment link generated",
  "data": {
    "collectRef": "<string>",
    "transactionId": "<string|null>",
    "paymentUrl": "<string|null>",
    "intentUrl": "<string|null>",
    "hostedUrl": "<string|null>",
    "paymentOptions": []
  }
}

Exact data depends on NSDL/NineteenPay response.

Example failure (no callback)

{
  "status": false,
  "respCode": 4001,
  "respMessage": "callbackUrl is required. Pass in request body or configure for your API key."
}

2) Status by collect reference

POST https://connect.striveedgepaints.com/api/nsdl-np/status

Headers — same as section 1 (Content-Type, Accept, X-API-Key).

Body

{
  "collect_ref": "YOUR_COLLECT_REF_FROM_INITIATE"
}
FieldRequired
collect_refYes (max 64 chars)

Success response (HTTP 200)

{
  "status": true,
  "respCode": 2000,
  "respMessage": "Transaction status fetched successfully",
  "paid": false,
  "data": {}
}

paid is boolean; data follows upstream status payload.

3) Link details (batch CRNs)

POST https://connect.striveedgepaints.com/api/nsdl-np/link-details

Headers — same as section 1.

{
  "crn_arr": ["CRN12345", "CRN67890"]
}
FieldRequired
crn_arrYes — non-empty array of strings (max 64 each)

Success response (HTTP 200)

{
  "status": true,
  "respCode": 2000,
  "respMessage": "OK",
  "data": {}
}

5) Provider webhook (inbound — NSDL/NineteenPay)

Not under /api/. Register this URL with the payment provider.

POST https://connect.striveedgepaints.com/webhook/nsdl-np

Headers

HeaderWhen
Content-Typeapplication/json
X-TSP-TimestampIf NSDL_NP_WEBHOOK_SECRET is set in Laravel
X-TSP-SignatureHMAC-SHA256 hex of timestamp + '.' + raw_body with webhook secret

If NSDL_NP_WEBHOOK_SECRET is empty, signature headers are not required.

Example body

{
  "collectRef": "CRN12345",
  "transactionId": "TXN_SIMULATED",
  "status": "SUCCESS",
  "requestAmount": "10.50",
  "utr": "405812345678"
}

Typical success (HTTP 200)

{
  "success": true,
  "message": "Notification received"
}

Postman — production variables

  1. Open the collection → Variables.
  2. Set middleware_base_url = https://connect.striveedgepaints.com
  3. Set middleware_api_key = your key
  4. Run requests 1 → 4 in order; copy collectRef from initiate into status/link-details.

Related server configuration (.env)

VariableRole
API_ALLOWED_IPSIPs allowed to call /api/nsdl-np/*
NSDL_NP_PROXY_URLLaravel calls WordPress proxy instead of NineteenPay directly
NSDL_NP_KEY / NSDL_NP_SALTDirect mode to nineteenapis (when not using proxy)
NSDL_NP_WEBHOOK_SECRETValidates inbound webhooks (X-TSP-*)

Integration reference — behavior follows routes/api.php, NsdlNpController, ApiKeyAuth, IpWhitelist, and WebhookController.
